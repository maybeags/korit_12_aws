# S3 이해하기
Backend 서버에서 이미지나 문서 같은 파일을 저장하고 관리하는 안정적인 저장소가 요구될 수 있습니다. AWS에서는 이런 '파일 저장소'로 S3를 많이 활용합니다.

## S3
AWS에서 제공하는 파일 저장 서비스 이름. 폰으로 사진 찍으면 구글 드라이브나 아이클라우드에 저장되는데, 그거랑 유사한 개념입니다.
1. 버킷(Bucket) : 구글 드라이브에서 공유 드라이브를 만들 수 있는 것처럼 S3에서도 저장소를 여러 개 만들 수 있습니다. 이렇게 만들어진 저장소를 버킷(bucket)이라고 합니다.
2. 객체(Object) : S3에서는 버킷에 업로드한 **파일을 객체** 라고 합니다. 이 객체는 key-value pair로 이루어져있습니다. 키는 객체에 할당한 이름, 값은 업로드한 컨텐츠 자체를 의미합니다. 객체의 값은 바이트 형태로 고정.

## S3를 사용하는 이유
보편적으로 사용되는 분야는 이미지 업로드 기능. 업로드된 이미지 파일은 백엔드 서버가 실행되는 EC2 인스턴스 내부에 저장하는 것도 가능하지만, 파일 개수가 많아지면 관리가 어렵습니다. 파일 저장할 공간이 한정되어있고, RDS 때와 마찬가지로 EC2 인스턴스에 이상이 생길 경우 데이터 손실 문제도 있기 때문입니다.
- S3는 파일 용량에 제한이 없고 사용자의 필요에 따라 자동으로 확장됩니다. 또한 데이터를 여러 물리적 위치에 분산하여 저장하기 때문에 데이터 손실 확률이 0.000000001 %로 희박합니다.
